# Local Docker Development Environment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run simple-feed-reader locally in Docker (nginx + PHP-FPM 8.3 + MySQL 8.4 + Mailpit, HTTPS via mkcert) without touching the native SQLite workflow.

**Architecture:** One `docker-compose.yml` at the repo root, build/config files under `docker/`. Compose injects `DATABASE_URL`/`MAILER_DSN` as real environment variables, which override the committed `.env` values inside containers only (Symfony env precedence). Spec: `docs/superpowers/specs/2026-07-22-local-docker-design.md`.

**Tech Stack:** Docker Compose, `php:8.3-fpm-alpine` + `install-php-extensions`, `nginx:1.27-alpine`, `mysql:8.4`, `axllent/mailpit`, mkcert, Xdebug 3 (trigger mode).

**Constraints for every task:**
- PHP platform is pinned to 8.3 — this environment exists partly to enforce that. Never install PHP 8.4.
- Docker is *additive*: no committed `.env`, `.env.test`, or `config/` file may change. If a task seems to require it, STOP and report BLOCKED.
- All `docker compose` commands run from the repo root. All host-side backend commands run from `backend/`.
- Commit after each task.

**Preflight (controller, before Task 1):** `docker version` must succeed and `mkcert -help` must exist (`brew install mkcert`). If either is missing, stop and ask the user to install it — do not install system tools yourself.

### Task 1: Compose skeleton — MySQL + Mailpit

**Files:**
- Create: `docker-compose.yml` (repo root)
- Create: `docker/mysql/init.sql`
- Modify: `.gitignore` (repo root)

- [ ] **Step 1: Create `docker/mysql/init.sql`**

The MySQL image auto-creates `feedreader` and grants `feedreader` rights on it, but Doctrine's `when@test` `dbname_suffix: '_test'` makes in-container test runs hit `feedreader_test`. This script (run once by the image's init hook, as root) creates the second database and the grant:

```sql
-- Runs only on first initialization of the named volume (docker-entrypoint-initdb.d).
-- The test database mirrors what Doctrine's dbname_suffix ('_test' in when@test)
-- derives from the dev DATABASE_URL, so one URL serves both dev and phpunit.
CREATE DATABASE IF NOT EXISTS feedreader_test CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
GRANT ALL PRIVILEGES ON feedreader_test.* TO 'feedreader'@'%';
FLUSH PRIVILEGES;
```

- [ ] **Step 2: Create `docker-compose.yml`**

```yaml
# Local development stack. Additive to the native workflow: nothing in here
# modifies committed .env files — container-only environment variables override
# them via Symfony's env precedence. Spec:
# docs/superpowers/specs/2026-07-22-local-docker-design.md

# Pin the project name so container/volume identity does not depend on the
# checkout location (a worktree directory would otherwise derive e.g. "docker").
name: simple-feed-reader

services:
  mysql:
    image: mysql:8.4
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: feedreader
      MYSQL_USER: feedreader
      MYSQL_PASSWORD: feedreader
    volumes:
      - mysql-data:/var/lib/mysql
      - ./docker/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    ports:
      # Non-standard host port so a natively installed MySQL never collides.
      - "33306:3306"
    healthcheck:
      test: ["CMD-SHELL", "mysqladmin ping -h 127.0.0.1 -uroot -p\"$$MYSQL_ROOT_PASSWORD\" --silent"]
      interval: 5s
      timeout: 3s
      retries: 20

  mailpit:
    image: axllent/mailpit:latest
    ports:
      - "8025:8025"   # web inbox; SMTP (1025) stays internal to the compose network

volumes:
  mysql-data:
```

- [ ] **Step 3: Git-ignore the cert directory**

Append to the repo-root `.gitignore`:

```
# Local TLS material generated by mkcert (docs/local-docker.md) — never commit keys.
/docker/certs/
```

- [ ] **Step 4: Validate and start**

Run: `docker compose config -q && docker compose up -d mysql mailpit`
Expected: exit 0, both containers start.

Run: `docker compose ps --format '{{.Name}} {{.Health}}'` (repeat until healthy; give MySQL ~30 s on first init)

Run: `docker compose exec mysql mysql -ufeedreader -pfeedreader -e "SHOW DATABASES;"`

### Task 2: PHP-FPM image with Xdebug

**Files:**
- Create: `docker/php/Dockerfile`
- Create: `docker/php/conf.d/app.ini`
- Create: `docker/php/conf.d/xdebug.ini`
- Modify: `docker-compose.yml` (add `php` service)

- [ ] **Step 1: Create `docker/php/Dockerfile`**

```dockerfile
# Dev image only. A future containerized production would add a multi-stage
# `prod` target here (composer install --no-dev, baked-in source, tuned opcache)
# — see the spec's "Deliberately out of scope" section.
FROM php:8.3-fpm-alpine

# install-php-extensions resolves each extension's build- and runtime-deps on
# alpine (icu for intl, libzip for zip, phpize toolchain for the xdebug pecl
# build) and cleans the build deps up afterwards.
COPY --from=mlocati/php-extension-installer /usr/bin/install-php-extensions /usr/local/bin/
RUN install-php-extensions pdo_mysql intl opcache zip xdebug

COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

COPY conf.d/app.ini /usr/local/etc/php/conf.d/app.ini
COPY conf.d/xdebug.ini /usr/local/etc/php/conf.d/xdebug.ini

WORKDIR /app
```

- [ ] **Step 2: Create `docker/php/conf.d/app.ini`**

```ini
; Dev-friendly limits; production values are a prod-stage concern.
memory_limit = 512M
```

- [ ] **Step 3: Create `docker/php/conf.d/xdebug.ini`**

```ini
; Trigger mode: requests run at full speed unless the IDE/browser sends the
; XDEBUG_TRIGGER cookie/param/env var, so the debugger costs nothing when idle.
xdebug.mode = debug
xdebug.start_with_request = trigger
; Docker Desktop resolves this to the macOS host, where the IDE listens on 9003.
xdebug.client_host = host.docker.internal
```

- [ ] **Step 4: Add the `php` service to `docker-compose.yml`**

Insert between `mysql` and `mailpit` (order is cosmetic):

```yaml
  php:
    build: ./docker/php
    volumes:
      - ./backend:/app
    environment:
      # Real env vars beat .env/.env.test file values (Symfony precedence), so
      # committed files stay untouched and the native SQLite workflow is
      # unaffected. In the test env Doctrine's dbname_suffix appends '_test',
      # so this one URL also points phpunit at feedreader_test.
      DATABASE_URL: "mysql://feedreader:feedreader@mysql:3306/feedreader?serverVersion=8.4&charset=utf8mb4"
      MAILER_DSN: "smtp://mailpit:1025"
    extra_hosts:
      # Makes host.docker.internal work on Docker Engine variants where it is
      # not built in; harmless on Docker Desktop.
      - "host.docker.internal:host-gateway"
    depends_on:
      mysql:
        condition: service_healthy
```

- [ ] **Step 5: Build and verify the toolchain**

Expected: build succeeds, container running.

Run: `docker compose exec php php -v`

Run: `docker compose exec php php -m | grep -Ei 'pdo_mysql|intl|zip|xdebug'`

Run: `docker compose exec php php -i | grep -E 'xdebug.mode|xdebug.start_with_request|xdebug.client_host'`

### Task 3: nginx with mkcert HTTPS

**Files:**
- Create: `docker/nginx/default.conf`
- Modify: `docker-compose.yml` (add `nginx` service)
- Create (generated, git-ignored): `docker/certs/localhost.pem`, `docker/certs/localhost-key.pem`

- [ ] **Step 1: Generate the locally-trusted certificate**

```bash
mkdir -p docker/certs
mkcert -cert-file docker/certs/localhost.pem -key-file docker/certs/localhost-key.pem localhost 127.0.0.1 ::1
```

Then run: `git status --short docker/`
Expected: certs do NOT appear (Task 1's `.gitignore` entry). If they appear, STOP — fix the ignore before anything else.

- [ ] **Step 2: Create `docker/nginx/default.conf`**

```nginx
# HTTPS is not optional locally: the __Host-oauth_flow cookie is Secure, and
# Safari (unlike Chrome) grants no Secure-cookie leniency to http://localhost.
server {
    listen 80;
    # Hardcoding the published port is fine for a local-only server block.
    return 301 https://localhost:8443$request_uri;
}

server {
    listen 443 ssl;
    http2 on;
    server_name localhost;

    ssl_certificate     /etc/nginx/certs/localhost.pem;
    ssl_certificate_key /etc/nginx/certs/localhost-key.pem;

    root /app/public;

    location / {
        try_files $uri /index.php$is_args$args;
    }

    location ~ ^/index\.php(/|$) {
        fastcgi_pass php:9000;
        fastcgi_split_path_info ^(.+\.php)(/.*)$;
        include fastcgi_params;
        # $realpath_root works because php and nginx mount ./backend at the
        # same path (/app), so resolved script paths agree between containers.
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        fastcgi_param DOCUMENT_ROOT $realpath_root;
        internal;
    }

    # Only index.php is a valid entrypoint; any other .php request is a probe.
    location ~ \.php$ {
        return 404;
    }
}
```

- [ ] **Step 3: Add the `nginx` service to `docker-compose.yml`**

```yaml
  nginx:
    image: nginx:1.27-alpine
    volumes:
      - ./backend:/app:ro
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
      - ./docker/certs:/etc/nginx/certs:ro
    ports:
      - "8080:80"
      - "8443:443"
    depends_on:
      - php
```

- [ ] **Step 4: Start and verify TLS + redirect**

Run: `docker compose up -d nginx`

Run: `curl -sS -o /dev/null -w '%{http_code} %{redirect_url}\n' http://localhost:8080/api/health`
Expected: `301 https://localhost:8443/api/health`

Run: `curl -sS -o /dev/null -w '%{http_code}\n' https://localhost:8443/api/health`
Expected: an HTTP status from PHP (200 or 500 — the DB has no schema yet; Task 4 fixes that). Must NOT be a curl TLS error: mkcert's root is in the macOS trust store, so no `-k` flag is needed or allowed here — `-k` passing while plain curl fails would mean the cert is wrong.

- [ ] **Step 5: Commit**

```bash
git add docker/nginx/default.conf docker-compose.yml
git commit -m "feat(docker): nginx tls termination with mkcert certificate"
```
